Lesson 2 Reference Code

Text Elements

<!-- HEADINGS: Six levels of hierarchy -->
<h1>Main Title — Use Once Per Page</h1>
<h2>Section Heading</h2>
<h3>Subsection Heading</h3>
<h4>…and so on down to H6</h4>

<!-- PARAGRAPHS: Block-level text container -->
<p>
  This is a paragraph of text. It's a block element,
  so it takes up the full width and creates space above/below.
</p>

<!-- TEXT FORMATTING: Inline elements that apply meaning -->
<p>
  Use <strong>strong</strong> for important text (screen readers emphasize),
  <em>em</em> for stressed emphasis,
  <mark>mark</mark> for highlighted text,
  and <code>code</code> for inline code snippets.
</p>

<!-- LINE BREAKS: Forces new line without new paragraph -->
<p>First line<br>Second line</p>

<!-- SPAN: Generic inline container, no semantic meaning -->
<p>You can style <span>specific words</span> with span.</p>

Key Concepts:

  • Block Elements - Take full width, start on new line (h1-h6, p, div)
  • Inline Elements - Only take space they need, flow with text (strong, em, span, code)
  • Heading Hierarchy - Always use h1 first, then h2, then h3 (don't skip levels)
  • Semantic HTML - Use <strong> (not <b>), <em> (not <i>) for better accessibility
